# HTTP狀態

## API應該使用多少個狀態代碼？
歸結為一個事實，在應用程序和API之間的交互中實際上只有3個結果：

* 一切正常
* Client做錯了
* API做錯了  


## Http Response Code
| 狀態碼 | 含意                              |
|-------|----------------------------------|
| 200   | 成功的請求和響應。(GET、DELETE、PUT) |
| 201   | 成功的請求和響應。(POST)            |
| 400   | 參數格式錯誤或其他錯誤請求。          |
| 401   | 需要授權以回應請求。                |
| 403   | 用戶端並無訪問權限。                |
| 404   | 伺服器找不到請求的資源。             |
| 500   | 伺服器端發生未知或無法處理的錯誤。     |
